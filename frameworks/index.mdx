## Overview

WebMCP works with any JavaScript framework. While we provide first-class React support via `@mcp-b/react-webmcp`, you can integrate WebMCP into Vue, Svelte, Angular, Rails, and any other framework using the vanilla JavaScript API.

This section covers framework-specific patterns, gotchas, and best practices for popular web frameworks.

## Integration Pattern

All non-React frameworks follow the same basic pattern:

<Steps>
  <Step title="Import the global API">
    Add `@mcp-b/global` to expose `navigator.modelContext`
  </Step>

  <Step title="Register tools using vanilla JS">
    Use `navigator.modelContext.registerTool()` directly
  </Step>

  <Step title="Handle cleanup">
    Call `unregister()` when components unmount
  </Step>

  <Step title="(Optional) Connect MCP client">
    For AI framework integration, connect an MCP client
  </Step>
</Steps>

## Basic Example

Here's the core pattern that works in any framework:

```typescript "Basic Tool Registration"
import '@mcp-b/global';

// Register a tool
const registration = navigator.modelContext.registerTool({
  name: 'get_user_data',
  description: 'Get the current user information',
  inputSchema: {
    type: 'object',
    properties: {
      includeEmail: {
        type: 'boolean',
        description: 'Include email in response'
      }
    }
  },
  async execute(args) {
    const user = getCurrentUser();
    return {
      content: [{
        type: 'text',
        text: JSON.stringify({
          name: user.name,
          email: args.includeEmail ? user.email : undefined
        })
      }]
    };
  }
});

// Later, when the component unmounts
registration.unregister();
```

## Framework Comparison

| Framework | Reactivity | Server Rendering | Tool Registration |
|-----------|------------|------------------|-------------------|
| Vue 3 | `ref`/`reactive` | Via Nuxt | `onMounted`/`onUnmounted` |
| Nuxt 3 | Vue reactivity | Built-in SSR | Client-only plugins |
| Svelte | `$state`/stores | Via SvelteKit | `onMount`/`onDestroy` |
| SvelteKit | Svelte stores | Built-in SSR | Client-side only |
| Rails | Stimulus | Turbo/Hotwire | Controller lifecycle |
| Angular | RxJS/Signals | Universal | Service lifecycle |

## Key Considerations

### Client-Side Only

WebMCP tools must be registered on the client side. If your framework uses server-side rendering (SSR), ensure tool registration only happens in the browser:

```typescript
// Check for browser environment
if (typeof window !== 'undefined' && 'modelContext' in navigator) {
  navigator.modelContext.registerTool({ /* ... */ });
}
```

### Cleanup is Critical

Always unregister tools when components are destroyed to prevent memory leaks and stale tool references:

```typescript
// Store the registration
const registration = navigator.modelContext.registerTool({ /* ... */ });

// On component destroy
registration.unregister();
```

### State Synchronization

When tools interact with framework state, ensure proper reactivity:

```typescript
// Tool handler should access current state, not stale closures
execute: async (args) => {
  // Access reactive state here
  const currentValue = myReactiveStore.value;
  return { content: [{ type: 'text', text: String(currentValue) }] };
}
```
