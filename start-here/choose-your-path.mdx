## By package

The `@mcp-b` packages are layered. Each builds on the one below it. Pick the layer that matches your needs.

<Steps>
  <Step title="Types only: @mcp-b/webmcp-types">
    Zero-runtime TypeScript types for `navigator.modelContext`. Install as a dev dependency for compile-time safety without adding any code to your bundle.

    ```bash
    npm install --save-dev @mcp-b/webmcp-types
    ```

    [Reference](/reference/runtime/webmcp-types)
  </Step>
  <Step title="Strict core polyfill: @mcp-b/webmcp-polyfill">
    Installs the WebMCP surface on `navigator.modelContext` without MCP-B extensions. Use this first when you want the browser standard in browsers today.

    ```bash
    npm install @mcp-b/webmcp-polyfill
    ```

    [Reference](/reference/runtime/webmcp-polyfill)
  </Step>
  <Step title="Full runtime: @mcp-b/global">
    Adds the MCP-B runtime layer on top of the core. Use this when you need transport, `listTools()`, `callTool()`, prompts, resources, or browser-to-desktop connectivity.

    ```bash
    npm install @mcp-b/global
    ```

    [Reference](/reference/runtime/global) | [Choose a runtime](/how-to/choose-runtime)
  </Step>
  <Step title="React hooks: @mcp-b/react-webmcp or usewebmcp">
    **`@mcp-b/react-webmcp`** provides hooks for the full MCP-B runtime, including provider and client patterns, Zod schema support, and execution state tracking.

    **`usewebmcp`** provides a single hook for strict core `navigator.modelContext` tool registration. No MCP extensions.

    ```bash
    # Full MCP-B hooks
    npm install @mcp-b/react-webmcp

    # Strict core hooks
    npm install usewebmcp
    ```

    [react-webmcp reference](/reference/runtime/react) | [Framework guide](/how-to/frameworks)
  </Step>
</Steps>

<Note>
  Not sure which package to pick? See [Choose a runtime](/how-to/choose-runtime) for a detailed comparison, or read [Strict core vs MCP-B extensions](/explanation/strict-core-vs-mcp-b-extensions) to understand the boundary.
</Note>
