## Package layering

The runtime packages form a strict dependency chain. Start with the standard path, then add MCP-B only if you need more than site-exposed tools. For a full explanation of why packages are split this way, see [Runtime Layering](/explanation/architecture/runtime-layering) and [Strict Core vs MCP-B Extensions](/explanation/strict-core-vs-mcp-b-extensions).

```
@mcp-b/global           MCP-B runtime layer
    |
@mcp-b/webmcp-ts-sdk    MCP SDK adapter (BrowserMcpServer)
    |
@mcp-b/webmcp-polyfill  Strict WebMCP polyfill
    |
@mcp-b/webmcp-types     Zero-runtime type definitions
    |
Native browser API       navigator.modelContext (Chromium)
```
